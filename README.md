# WWW Applications — Laboratory Project

## Overview

This repository contains the coursework project for **Laboratories 1–3** in the *WWW Applications* subject.  
The project incrementally evolves from a minimal HTTP server to a structured, styled multi-page website.

---

## Laboratory 1 — Minimal HTTP Server

**Objective**  
Implement a basic HTTP server in Python running on `localhost:8000`.

**Core Requirements**

- Listen for incoming TCP connections
- Parse raw HTTP request text manually
- Generate valid HTTP responses
- Maintain a persistent counter of total server accesses

**Outcome**

A functional low-level HTTP server demonstrating understanding of:

- Socket programming
- HTTP protocol structure
- Request/response lifecycle
- Stateful persistence

---

## Laboratory 2 — Multi-Page HTML Website

**Objective**  
Extend the Lab 1 server to serve a structured, multi-page HTML website.

**Core Requirements**

- Serve multiple HTML documents
- Implement internal navigation via hyperlinks
- Create and process HTML forms
- Use semantic HTML elements properly
- Display structured data using tables

**Outcome**

A semantically correct, navigable website served by a custom-built HTTP server.

---

## Laboratory 3 — CSS Styling and Layout

**Objective**  
Introduce CSS to transform the website into a visually consistent, responsive interface.

**Core Requirements**

- Separate structure (HTML) from presentation (CSS)
- Eliminate all inline styles
- Implement responsive layout techniques
- Ensure visual consistency across all pages

**Outcome**

A cleanly structured website with centralized styling and layout fully controlled through CSS.
